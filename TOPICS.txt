4. SESSION ARCHITECTURE
------------------------

4.1 TcpSession / WsSession
    - Represents one client connection
    - Owns socket
    - Owns read buffer
    - Owns write queue

4.2 Lifetime Management
    - shared_from_this()
    - Self-capturing handlers
    - Prevent premature deletion

4.3 Read Loop Pattern
    - async_read → handler → re-arm read

4.4 Write Queue Pattern
    - push to queue
    - if not writing → start async_write
    - on write completion → pop + continue

7. ASYNC FLOW (WHAT ACTUALLY HAPPENS)
--------------------------------------

7.1 Accept Flow
    async_accept()
        ↓
    Create Session
        ↓
    start()
        ↓
    async_read()

7.2 Message Flow
    Client sends JSON
        ↓
    async_read handler
        ↓
    Parse JSON
        ↓
    Route to RoomManager
        ↓
    ChatRoom broadcast()
        ↓
    async_write to each session

7.3 Write Flow
    queue.push(message)
        ↓
    if idle → async_write
        ↓
    completion handler
        ↓
    queue.pop()
        ↓
    continue if more
